Iniciando o projeto

* Antes de começar o projeto é necessário entrr na branch develop e dar o pull para que os arquivs fiquem atualizados, assim evitando conflitos.

- Super usuário

para todos os comandos é necessário usar o super-usuário do Linux: sudo seguido do comando git

- Entrando na branch

Para acessar a branch referente ao ambiente de desenvolvimento é necessário rodar o comando:

sudo git checkout develop
- Sincronizando alterações

Para evitar conflitos e garantir que o projeto esteja atualizado, é necessario rodar o comando:

sudo git pull prigin develop

* Branch

Bom, para mexer com o flow é necessário se referir algumas vezes a branch que está sendo feito o commit. A branch contém algumas informações cruciais como o número da issue (demanda), nome do desenvolvedor, data do commit, etc.

ela segue o seguinte padrão:

número da demanada_nome do usuário no GitHub_titulo da branch, ele deve ser um pouco descritivo porém sucinto e não pode contere caracteres especiais ou espaços_data do commit no formato: número do mês_ano

Num_nomeDev_titulo_NumMes_Ano

Flow

Antes do add é preciso usar o flow a depender do tipo de implementação feita

- Feature

Caso a implementação seja referente a uma feature (nova funcionalidade) é necessário rodar esse comando:

feature start *Branch
- Bugfix

Caso a implementação seja referente a um bug (consertar bug) é necessário rodar esse comando:

bugfix start *Branch
- Hotfix

Caso a implementação seja referente a uma hotfix (correção rápida para um problema crítico) é necessário rodar esse comando:

hotfix start *Branch

* nesse caso há mais um passo ao fim de tudo, é preciso finalizar (finish)

Commitando alterações

para realizar commits é necessário dois passos.

- Adicionando alterações

Para adicionar as alterações basta rodar o comando:

sudo git add .
- Realizando commit

Ao realizar o commit é possível especificar melhor o que foi implementado com o comando:

sudo git commit -m "#Num da issue_tipo da branch | texto"

Publicando alterações

para publicar suas alterações basta rodar o comando:

sudo git flow tipo_da_branch publish *Branch
- Finalizando hotfix

*Caso o tipo da branch seja hofix, será necessário finalizar com esse comando após a publicação:

sudo git flow hotfix finish *Branch
